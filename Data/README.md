# 모집결과·요강 데이터 배치 규칙

이 문서는 `Data/` 아래에 **모집결과·요강 원본을 어떻게 배치해야 추천 품질이 안정적으로 유지되는지** 설명합니다. 현재 프로젝트는 별도 등록 DB 없이 **경로 이름만으로 지역·학교 힌트**를 추론하므로, 폴더 구조 자체가 데이터 품질의 일부입니다.

## 왜 경로에 권역·시도·시군을 넣나

- 별도 "파일 등록 DB" 없이 사용자가 선택한 지역과 디스크 상 파일을 연결하려면, 경로 문자열에 **검색 가능한 힌트**가 있어야 합니다.
- 초기 제품 단계에서는 등록 UI와 스키마 마이그레이션 부담을 줄이기 위해, **경로 기반 매칭**을 우선 채택했습니다.
- `영남권 / 경북 / 경산시 / 대학교명`처럼 단계가 드러나면, 코드가 광역 지역을 시도 단위로 풀고 학교 후보를 더 좁게 잡을 수 있습니다.

이 방식의 한계는 **폴더 구조가 틀리면 필터도 흔들린다**는 점입니다. 매칭이 자주 어긋나면 `region_hints` 매핑을 보강하거나, 아예 별도 메타데이터 테이블을 도입하는 편이 낫습니다.

## 권장 디렉터리 구조

```text
Data/
  영남권/
    경북/
      {시·군명}/
        {대학교명}/
          {연도}_{수시|정시}_…_모집결과|입시결과.xlsx
          (또는 동일 조건의 .pdf)
    경남/
      …
    부산/
      …
  수도권/
    경기/
      …
```

- **1단계 `영남권`·`수도권` 등**: 사용자 선택과 직접 연결되는 권역 태그
- **2단계 `경북`·`경남`·`서울` 등**: 짧은 시·도 토큰
- **3단계 시·군**: `경산시`, `구미시`처럼 행정구역명
- **4단계 학교 폴더**: `대구대학교`, `영남대학교` 등 공식 학교명

가능하면 **사람이 폴더만 봐도 출처를 바로 이해할 수 있는 이름**을 유지하는 것이 좋습니다.

## 엑셀(xlsx·xls)과 PDF

- **OpenAI 추천 경로:** 모집결과 `xlsx`·`xls`·`pdf`를 모두 `input_file`로 활용할 수 있습니다.
- **후보 선택 로직:** 같은 학교·연도·수시/정시 묶음에 엑셀과 PDF가 같이 있으면, 일반적으로 **엑셀만** 후보로 남깁니다.
- **인제스션:** 현재 파이프라인은 **스프레드시트만** parquet로 정규화합니다. PDF는 테이블 캐시로 변환되지 않지만, 추천 시점의 원문 후보에는 포함될 수 있습니다.

### 왜 "엑셀만 인제스션"이고 PDF는 후보만인가

- **엑셀 → parquet**는 재현 가능한 파이프라인을 만들기 쉽고, 컬럼 단위 후처리에도 유리합니다.
- **PDF → parquet**는 레이아웃, 다단, 스캔본 여부에 따라 품질 편차가 커서 구조화 캐시로 쓰기 어렵습니다.
- 그래서 이 프로젝트는 **정확도가 중요한 구조화 층은 스프레드시트**, **원문 참조는 PDF**로 역할을 나눕니다.

## `dataset_id`가 경로 기반인 이유와 트레이드오프

- 파일이 이동하면 **새 데이터셋 ID**로 취급됩니다. 이는 `경로 = 출처 식별자`라는 단순화를 택한 결과입니다.
- 장점은 구현이 단순하고, 사람이 폴더만 봐도 출처를 이해하기 쉽다는 점입니다.
- 단점은 폴더를 옮길 때마다 ingestion을 다시 돌려야 한다는 점입니다.

## 이미지(.png 등)

**추천 LLM 파일 목록에는 넣지 않습니다.** 공지가 이미지뿐이라면, 가능하면 PDF·엑셀 원본을 구하거나 OCR 후 검수한 파일로 바꾸는 편이 좋습니다.

## 파일 이름 권장 규칙

- **`모집결과`·`입시결과`·`전형결과`·`충원`·`합격`·`경쟁률`** 같은 키워드는 분류 정확도를 높입니다.
- **`{학교명}_{연도}_{수시|정시}_...`** 형태로 시작하면 검색, 필터, 중복 제거에 유리합니다.

## `4년재`·`4년제` 폴더로 받은 자료 합치기

저장소 루트에서:

```bash
python scripts/reorganize_gyeongbuk_import.py
```

`Data/4년재/4년재/`(또는 `4년재/`) 내용을 `Data/영남권/경북/` 아래로 옮기고, 동일 경로·동일 내용(SHA-256)은 삭제합니다. 같은 폴더 안에서 파일명만 다른 동일 파일이 있으면 **엑셀을 우선**해 하나만 남깁니다.

## 경로를 바꾼 뒤

`dataset_id`가 경로 기반이므로, 폴더나 파일 구조를 바꾼 뒤에는 **ingestion을 다시 실행**해 카탈로그를 갱신해야 합니다.

`storage/`(카탈로그·parquet)는 저장소에 포함되지 않습니다. 새 환경에서는 `Data/`를 맞춘 뒤 **`POST /api/v1/ingestion/run`** 또는 [backend/README.md](../backend/README.md)의 로컬 `python -c` 예시로 수집을 실행하세요.
